## 三、TypeScript 类型安全评估

### 3.1 当前状态 ⚠️ 需改进

**优点：**
- ✅ 有基础的类型定义 (`types/`)
- ✅ 使用泛型 `ApiResponse<T>`
- ✅ 接口定义较完整

**问题：**

| # | 问题 | 严重性 | 位置 |
|---|------|--------|------|
| 1 | **大量 `any` 类型** | 高 | `CrudTable.vue:81` - `data: any[]` |
| 2 | **`@ts-nocheck` 注释** | 高 | `CrudTable.vue:67` |
| 3 | **类型定义不完整** | 中 | `dispatchCommand(data: any)` |
| 4 | **混合风格命名** | 低 | snake_case vs camelCase |
| 5 | **缺少泛型约束** | 中 | 多个 API 函数 |

### 3.2 问题代码示例

```typescript
// ❌ 问题代码 - 使用 any
const props = defineProps<{
  data: any[];  // 丢失类型信息
  columns: Column[];
}>();

// ✅ 改进代码
interface Props<T extends Record<string, any>> {
  data: T[];
  columns: Column<T>[];
}

// ❌ 问题代码 - @ts-nocheck
// @ts-nocheck  // 禁用类型检查，危险！

// ✅ 改进代码 - 移除并修复类型
interface Column<T> {
  prop: keyof T;
  label: string;
}
```

### 3.3 类型定义改进建议

```typescript
// 1. 统一命名风格 (建议 camelCase 与后端 snake_case 转换)
// types/vehicle.ts
export interface Vehicle {
  vehicleId: number;
  licensePlate: string;
  driverName?: string;
  // ...
}

// 2. API 响应类型统一
export interface ApiResponse<T> {
  code: number;
  message: string;
  data: T;
}

// 3. 分页类型
export interface PaginatedResponse<T> {
  items: T[];
  total: number;
  page: number;
  pageSize: number;
}
```

**代码质量示例 (`useAuthStore.ts`)：**
```typescript
export const useAuthStore = defineStore('auth', () => {
  // ✅ 使用 Composition API 风格
  const user = ref<User | null>(...);
  const is_authenticated = computed(() => !!user.value);

  // ✅ HttpOnly Cookie 方案（安全）
  const logout = async () => {
    await authApi.logout();
    user.value = null;
  };

  return { user, is_authenticated, logout, ... };
});
```

**问题：**

| 3 | **缺少 store 测试** | 中 | 未编写单元测试 |

### 5.2 状态管理改进建议

```typescript
// 1. 添加 Pinia 持久化
import { createPinia } from 'pinia';
import piniaPluginPersistedstate from 'pinia-plugin-persistedstate';

const pinia = createPinia();
pinia.use(piniaPluginPersistedstate);

// 2. 敏感信息不持久化
const useAuthStore = defineStore('auth', () => {
  const user = ref<User | null>(null);

  return { user }; // 不持久化，通过 session 恢复

// 3. 添加 store 测试
describe('useAuthStore', () => {

  });
```
