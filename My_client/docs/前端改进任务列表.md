## 五、关键改进建议

### 优先修复的测试失败
```
npm run test 显示 37 个协议测试失败，主要原因：
1. LegacyProtocolAdapter 返回格式与预期不符
2. NewProtocolAdapter 返回格式与预期不符
3. ProtocolManager 方法返回格式变更

建议：检查协议适配器实现，更新测试预期或修复实现
```

### 类型安全改进
```typescript
// 当前问题：api/index.ts 中存在 any 类型
// 改进方向：为每个 API 函数添加完整的请求/响应类型定义

// 示例改进：
interface Vehicle {
  vehicleId: number;
  licensePlate: string;
}

export const getVehicles = async (): Promise<Vehicle[]> => {
  const response = await api.get('/api/vehicles');
  return response.data;
};
```

### 性能优化重点
```
1. 将大数据列表页面（如车辆管理、订单列表）迁移到 VirtualTable
2. 添加图片懒加载到图片密集页面
3. 添加请求去重避免重复 API 调用
```
